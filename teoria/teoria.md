## ¿Qué es Markdown?
Markdown es un lenguaje de marcado ligero que permite dar formato a texto plano de manera sencilla mediante símbolos como #, *, -, entre otros. Es ampliamente utilizado en documentación técnica y repositorios de GitHub.

# GIT

## 1. ¿Qué es un repositorio en Git?
Un repositorio es un espacio donde se almacena el historial completo de cambios de un proyecto permitiendo el control de versiones.

## 2. Tres áreas principales de Git
- Working Directory: archivos del proyecto
- Staging Area: área de preparación
- Repository: historial permanente de commits

## 3. ¿Cómo representa Git los cambios?
Git utiliza objetos blob, tree, commit y tag para representar archivos, estructuras y versiones.

## 4. ¿Cómo se crea un commit?
Con el comando:

git commit -m "mensaje"

Guarda autor, fecha, mensaje y cambios realizados.

## 5. Diferencia entre git pull y git fetch
- fetch descarga cambios
- pull descarga y fusiona cambios

## 6. ¿Qué es un branch?
Es una rama de desarrollo independiente del proyecto principal.

## 7. ¿Cómo se realiza un merge?
Con git merge, uniendo ramas. Si hay conflictos deben resolverse manualmente.

## 8. ¿Cómo funciona git add?
Agrega cambios al área de staging antes del commit.

## 9. ¿Qué es .gitignore?
Archivo que indica qué archivos no deben rastrearse.

## 10. Commit amend vs nuevo commit
--amend modifica el último commit, mientras que un nuevo commit crea un registro adicional.

## 11. ¿Qué es git stash?
Permite guardar cambios temporales sin realizar commit.

## 12. Mecanismos para deshacer cambios
git reset, git revert y git checkout.

## 13. Configuración de remotos
origin es el repositorio principal y upstream el repositorio base en forks.

## 14. Inspeccionar historial
git log, git diff y git show.

# PROGRAMACIÓN

## 15. Tipos de datos primitivos en Java
byte, short, int, long, float, double, char y boolean.

## 16. Estructuras de control
if, else, switch, for, while y do-while.

## 17. Importancia de nombres significativos
Mejoran la legibilidad y mantenimiento del código.

## 18. ¿Qué es POO?
Paradigma basado en objetos que combinan datos y comportamiento.

## 19. Cuatro pilares de la POO
Encapsulamiento, abstracción, herencia y polimorfismo.

## 20. Herencia en Java
Permite que una clase herede atributos y métodos usando extends.

## 21. Modificadores de acceso
public, private, protected y default.

## 22. ¿Qué es una variable de entorno?
Es una variable del sistema que almacena configuraciones necesarias para ejecutar programas como Java.
